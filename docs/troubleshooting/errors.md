# Error Code Reference

Every cantonctl error includes a code, human-readable message, actionable suggestion, and a link back to this page. Error codes are organized by subsystem.

---

## E1xxx: Configuration

### E1001 — Config Not Found

**Message:** No cantonctl.yaml found in this directory or any parent directory.

**Symptoms:**
- Running any command outside a cantonctl project
- Running a command in a subdirectory where no parent has `cantonctl.yaml`

**Resolution:**
1. Run `cantonctl init <name>` to create a new project
2. Or `cd` into a directory that contains `cantonctl.yaml`
3. cantonctl searches upward from the current directory — ensure the file exists somewhere in the path

---

### E1002 — Invalid YAML

**Message:** cantonctl.yaml contains invalid YAML syntax.

**Symptoms:**
- Malformed YAML in `cantonctl.yaml` (bad indentation, missing colons, unquoted special characters)

**Resolution:**
1. Validate your YAML with a linter: `npx yaml-lint cantonctl.yaml`
2. Common issues: tabs instead of spaces, missing colon after key, unquoted strings starting with `{` or `[`
3. Compare against the template generated by `cantonctl init`

---

### E1003 — Schema Violation

**Message:** cantonctl.yaml does not match the expected schema.

**Symptoms:**
- Missing required fields (`version`, `project.name`, `project.sdk-version`)
- Invalid values (wrong type, invalid enum value)
- The error context includes the specific field paths and messages

**Resolution:**
1. Check the suggestion in the error output — it lists the exact fields that failed validation
2. Required fields: `version` (number), `project.name` (string), `project.sdk-version` (string)
3. Valid network types: `sandbox`, `remote`, `docker`
4. Valid party roles: `operator`, `participant`, `observer`
5. Valid auth methods: `jwt`, `shared-secret`, `none`

---

### E1004 — Directory Already Exists

**Message:** Target directory already exists.

**Symptoms:**
- Running `cantonctl init <name>` when a directory with that name already exists

**Resolution:**
1. Choose a different project name
2. Or remove the existing directory: `rm -rf <name>`

---

## E2xxx: SDK/Tools

### E2001 — SDK Not Installed

**Message:** Neither dpm nor daml CLI found on PATH.

**Symptoms:**
- Running `cantonctl dev`, `cantonctl build`, or `cantonctl test` without the Daml SDK installed

**Resolution:**
1. Install the Daml SDK: https://docs.daml.com/getting-started/installation.html
2. Or install dpm (Canton Package Manager): https://www.digitalasset.com/developers
3. Verify installation: `dpm version` or `daml version`
4. Ensure the binary is on your PATH

---

### E2002 — SDK Version Mismatch

**Message:** Installed SDK version is incompatible with this project.

**Symptoms:**
- The installed SDK version doesn't match `project.sdk-version` in `cantonctl.yaml`

**Resolution:**
1. Check the required version: `grep sdk-version cantonctl.yaml`
2. Install the correct version: `dpm install <version>` or update `cantonctl.yaml`

---

### E2003 — SDK Command Failed

**Message:** SDK command exited with a non-zero status.

**Symptoms:**
- A `dpm` or `daml` subcommand (build, test, codegen) exited with an error
- Git clone failed when using `--from` flag

**Resolution:**
1. Check the stderr output in the error context for the specific failure
2. For build errors, see E4001
3. For git clone failures, verify the URL and network connectivity

---

## E3xxx: Sandbox/Node

### E3001 — Sandbox Start Failed

**Message:** Canton sandbox process exited unexpectedly during startup.

**Symptoms:**
- `cantonctl dev` starts the sandbox process but it crashes immediately
- Often caused by missing SDK components or corrupt installation

**Resolution:**
1. Verify SDK installation: `dpm version`
2. Try starting sandbox manually: `dpm sandbox --port 5001 --json-api-port 7575`
3. Check if another sandbox is already running on the same ports (see E3002)
4. Reinstall the SDK if the binary is corrupt

---

### E3002 — Port In Use

**Message:** The requested port is already in use.

**Symptoms:**
- Another process is already listening on port 5001 or 7575
- A previous `cantonctl dev` session wasn't shut down cleanly

**Resolution:**
1. Find the process using the port: `lsof -i :5001` or `lsof -i :7575`
2. Kill it: `kill <PID>`
3. Or use different ports: `cantonctl dev --port 6001 --json-api-port 8575`

---

### E3003 — Health Timeout

**Message:** Canton sandbox did not become healthy within the timeout.

**Symptoms:**
- Sandbox process started but the JSON Ledger API never responded within 60 seconds
- The sandbox may be slow to start or may have crashed silently

**Resolution:**
1. Check if the ports are already in use (see E3002)
2. Check system resources (memory, CPU)
3. Try increasing the timeout or restarting
4. Check sandbox process logs for errors

---

## E4xxx: Build

### E4001 — Daml Compilation Error

**Message:** Daml compilation failed.

**Symptoms:**
- Syntax errors, type errors, or missing imports in `.daml` files
- During `cantonctl build` or hot-reload in `cantonctl dev`

**Resolution:**
1. Check the stderr output for specific compilation errors
2. Fix the Daml source code
3. In dev mode, the server continues running — fix the error and save again

---

### E4002 — DAR Not Found

**Message:** Expected .dar file was not produced by the build.

**Symptoms:**
- Build succeeded but no `.dar` file was found in `.daml/dist/`

**Resolution:**
1. Verify `daml.yaml` has correct `name` and `source` fields
2. Check that `daml/` directory contains valid Daml source files
3. Run `dpm build` manually to see detailed output

---

## E5xxx: Test

### E5001 — Test Execution Failed

**Message:** One or more Daml Script tests failed.

**Symptoms:**
- Test assertions failed or test scripts threw exceptions

**Resolution:**
1. Check the test output for which tests failed and why
2. Use `--filter` to run specific tests: `cantonctl test --filter myTest`

---

## E6xxx: Deploy

### E6001 — Deploy Auth Failed

**Message:** Authentication failed for the target network.

**Resolution:** Generate or refresh your JWT token for the target network.

### E6002 — Deploy Network Unreachable

**Message:** Cannot connect to the target Canton participant.

**Resolution:** Verify the network URL in `cantonctl.yaml` and your network connectivity.

### E6003 — Deploy Upload Failed

**Message:** DAR upload was rejected by the participant.

**Resolution:** Check the participant logs. The DAR may be incompatible or the participant may not accept uploads.

### E6004 — Deploy Package Exists

**Message:** A package with this name and version already exists.

**Resolution:** Increment the version in `daml.yaml` or use `--force` to overwrite.

---

## E7xxx: Ledger API

### E7001 — Ledger Connection Failed

**Message:** Cannot connect to the Canton JSON Ledger API.

**Resolution:** Verify the sandbox is running and the JSON API port is correct.

### E7002 — Ledger Command Rejected

**Message:** Command was rejected by the ledger.

**Resolution:** Check party authorization, template existence, and command payload.

### E7003 — Ledger Auth Expired

**Message:** JWT token has expired or is invalid.

**Resolution:** Generate a new token. For local development, `cantonctl dev` handles this automatically.

---

## E8xxx: Console/REPL

### E8001 — Console Parse Error

**Message:** Could not parse the console command.

**Resolution:** Check command syntax. Type `help` for available commands.

### E8002 — Console Unknown Command

**Message:** Unknown console command.

**Resolution:** Type `help` to see available commands.
